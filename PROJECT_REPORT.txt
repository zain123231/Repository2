1. EXECUTIVE SUMMARY
----------------------------------------------------------------------
This report documents the findings of a 100% test-coverage evaluation 
across 2,997 test images on four different variants of the GeoCLIP 
pipeline (A1, A2, A3, A4). We resolved system fragmentation by unifying 
legacy and modern codebases into a single, rigorous evaluation framework.

2. BASELINE A1 INTEGRITY (THE CONTROL)
----------------------------------------------------------------------
Baseline A1 (Single Crop, No Refinement) achieved the highest overall 
accuracy at the 1km radius (13.0%). This confirms that the native zero-shot 
CLIP backbone possesses strong intrinsic spatial understanding. No data 
leakage or contamination was found in the evaluation pipeline.

3. THE TTA PENALTY (VARIANT A2)
----------------------------------------------------------------------
Variant A2 introduced Test-Time Augmentation (10-Crop TTA). 
Surprisingly, TTA penalized the ultra-fine accuracy (Acc@1km dropped 
from 13.0% to 5.4%). The 10 crops average out distinct local features, 
which blurs the exact geographical signature needed for sub-kilometer 
predictions.

4. MICRO-GRID REFINEMENT (VARIANTS A3 & A4)
----------------------------------------------------------------------
Variants A3 and A4 applied dense micro-grid refinement over the coarse 
candidates. While the current evaluation improved the A3/A4 Acc@1km from 
0.5% (in the archived run) to 2.7% (in the current run) due to our recent 
algorithmic fixes, it still falls short of Baseline A1. 

5. ARCHITECTURE MISMATCH
----------------------------------------------------------------------
The failure of Micro-Grid to surpass A1 suggests an architectural mismatch:
Pre-trained CLIP models are not natively trained to distinguish 40x40 
micro-grids covering the same 50km radius. They struggle to differentiate 
between a street in Paris and the street one block over based solely on 
zero-shot textual/spatial embeddings.

6. OCR INTEGRATION (VARIANT A4)
----------------------------------------------------------------------
Variant A4 integrates EasyOCR. While OCR provides critical auxiliary cues 
(e.g., reading a street sign), its statistical impact on a random global 
dataset like Im2GPS3k is minimal because less than 5% of images contain 
legible, geolocatable text. However, its forensic value in the UI is 
immense.

7. CONFIDENCE CALIBRATION & UI OVERHAUL
----------------------------------------------------------------------
We discovered a bug where the `temperature` scaling parameter was ignored 
in the `argmax` mode of the Micro-Grid refinement. Fixing this, and 
applying a temperature of 0.35 in the Web UI (`app.py`), resulted in 
realistic, human-readable confidence scores. The UI was completely 
overhauled with Premium Glassmorphism CSS.

8. LEGACY SYSTEM UNIFICATION
----------------------------------------------------------------------
We successfully ported the advanced graphing logic (Heatmaps, CDF Plots) 
from the legacy `Repository2-main` scripts into the new rigorous evaluation 
system. The new pipeline now produces publication-quality charts using 
real evaluation data.

10. FUTURE RECOMMENDATIONS
----------------------------------------------------------------------
To make Micro-Grid Refinement viable, the location encoder must be 
fine-tuned with Contrastive Loss explicitly penalizing incorrect grid 
cells within the same city. Zero-shot CLIP is too coarse. For immediate 
applications, Baseline A1 should be used for automated benchmarking, 
while A4 should remain in the UI for interactive forensic analysis.
